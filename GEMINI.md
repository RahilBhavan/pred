# Gemini Configuration and Best Practices

## Operational Guidelines
- **Commit & Push:** After every major update or feature implementation, you MUST commit the changes with a clear, concise message and push them to the GitHub repository.
- **Directives vs Inquiries:** Treat all requests as Inquiries unless they contain an explicit instruction to perform a task.
- **Research first:** Always research the codebase before proposing or implementing changes.
- **Testing:** Every change must be accompanied by appropriate tests.

## Code Style
- Follow existing patterns in the codebase.
- Prefer TypeScript for web projects and Python for backend/data science.
